      40             : /// Definition of the integrator self-starting procedure.
      41             : ///
      42             : /// The self-start procedure generates \f$N\f$ function values of
      43             : /// accuracy \f$O\left((\Delta t)^N\right)\f$, where \f$N\f$ is the
      44             : /// negative of the initial slab number.  To generate values, it
      45             : /// requires the time stepper to be a multistep integrator that will
      46             : /// produce an order-\f$k\f$-accurate result given \f$k-1\f$ history
      47             : /// values.
      48             : ///
      49             : /// If the integrator is started from analytic history data or
      50             : /// requires no history (such as for a substep integrator), then the
      51             : /// initial slab number can be set to zero and no self-start steps
      52             : /// will be taken.
      53             : ///
      54             : /// \details
      55             : /// To self-start a multistep integrator, the function is integrated
      56             : /// repeatedly with increasing accuracy.  A first order integrator
      57             : /// (Euler's method) requires no history values, so it can be used,
      58             : /// starting from the initial point, to generate a
      59             : /// first-order-accurate value at a later time.  We then reset to the
      60             : /// start conditions and use the new "history" value (at a discarded
      61             : /// point later in time) to take two steps with a second-order method.
      62             : /// These values are second-order-accurate despite the history only
      63             : /// being first-order because the calculation of the change in the
      64             : /// value multiplies the previous derivatives by a factor of
      65             : /// \f$\Delta t\f$.  The time and value are then again reset to their
      66             : /// starting values and we start again at third order, and so on.
      67             : ///
      68             : /// The choice of performing the low-order integrations in the same
      69             : /// direction as the main integration makes this a _forward_
      70             : /// self-start procedure, as opposed to a _backward_ procedure that
      71             : /// produces values for times before the start time.  The primary
      72             : /// advantage of the forward version is that the solution is
      73             : /// guaranteed to exist after the start time, but not before.  It also
      74             : /// makes bookkeeping easier, as the reset after each order increase
      75             : /// is to the initial state, rather than to a time one step further
      76             : /// back for each order.  It does have the disadvantage, however, of
      77             : /// leaving a non-monotonic history at the end of the procedure, which
      78             : /// the main evolution loop must be able to handle.
      79             : ///
      80             : /// Each time the state is reset the slab number is increased by one.
      81             : /// This ensures that the evaluations are considered to be ordered in
      82             : /// their evaluation order, even though they are not monotonic in
      83             : /// time.  When the slab number reaches zero the initialization
      84             : /// procedure is complete and history appropriate for use for an
      85             : /// integrator of order \f$N+1\f$ has been generated.
      86             : ///
      87             : /// The self-start procedure performs all its evaluations before the
      88             : /// end of the first time step of the main evolution.  It is important
      89             : /// that none of the early steps fall at the same time as the
      90             : /// self-start history values, so the main evolution should not
      91             : /// decrease its step size on the first step after the procedure.
      92             : /// Additionally, the history times will not be monotonicly increasing
      93             : /// until \f$N\f$ steps have been taken.  The local-time-stepping
      94             : /// calculations require monotonic time, so local time-stepping should
      95             : /// not be initiated until the self-start values have expired from the
      96             : /// history.  These restrictions on step-size changing are checked in
      97             : /// the TimeStepper::can_change_step_size method.
