      64             :  * Creates a 3D Domain that represents a binary compact object solution. The
      65             :  * Domain consists of 4, 5, or 6 nested layers of blocks; these layers are,
      66             :  * working from the interior toward the exterior:
      67             :  * - 0: (optionally) The block at the center of each compact object, if not
      68             :  *      excised. If present, this block is a cube. If excised, the hole left
      69             :  *      by its absence is spherical.
      70             :  * - 1: The blocks that resolve each individual compact object. This layer has
      71             :  *      a spherical outer boundary - if the corresponding layer-0 block exists,
      72             :  *      then the layer is a cube-to-sphere transition; if the layer-0 block is
      73             :  *      excised, then the layer is a spherical shell.
      74             :  * - 2: The blocks that surround each object with a cube. Around each compact
      75             :  *      object, this layer transitions from a sphere to a cube.
      76             :  * - 3: The blocks that surround each cube with a half-cube. At this layer, the
      77             :  *      two compact objects are enclosed in a single cube-shaped grid.
      78             :  * - 4: The 10 blocks that form the first outer shell. This layer transitions
      79             :  *      back to spherical. The gridpoints are distributed linearly with respect
      80             :  *      to radius.
      81             :  * - 5: The 10 blocks that form a second outer shell. This layer is
      82             :  *      spherical, so a logarithmic map can optionally be used in this layer.
      83             :  *      This allows the domain to extend to large radial distances from the
      84             :  *      compact objects. This layer can be h-refined radially,
      85             :  *      creating a layer of multiple concentric spherical shells.
      86             :  *

      88             :  * compact objects, and by extension, also refer to the layers that immediately
      89             :  * surround each compact object. Note that `ObjectA` is located to the left of
      90             :  * the origin (along the negative x-axis) and `ObjectB` is located to the right
      91             :  * of the origin. `enveloping cube` refers to the outer surface of Layer 3.
